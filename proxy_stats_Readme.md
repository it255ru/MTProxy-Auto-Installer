# proxy-stats

Утилита мониторинга подключений для **telemt** MTProxy.  
Показывает активных пользователей, трафик и причины неудачных подключений в одном экране.

---

## Использование

```bash
# Разовый запуск
proxy-stats

# Автообновление каждые 10 секунд
watch -n 10 proxy-stats

# Автообновление каждые 30 секунд (меньше нагрузка)
watch -n 30 proxy-stats
```

## Расшифровка полей

### Активные пользователи

| Поле | Описание |
|---|---|
| `✓ online` | Есть активные подключения прямо сейчас |
| `— offline` | Нет активных подключений |
| `conns=N` | Количество открытых соединений от этого пользователя |
| `traffic=N MB` | Суммарный трафик с момента запуска telemt |
| IP-адреса | Все активные IP клиентов с геолокацией |

### Неудачные подключения

| Категория | Значение | Что делать |
|---|---|---|
| **Mobile operator blocking MTProto** | Клиент доходит до сервера, но DPI мобильного оператора обрывает MTProto-хэндшейк. Характерно для Билайна (Vimpelcom), отдельных регионов МТС. По WiFi обычно работает. | Ограничение на стороне оператора — не лечится. Попробовать другой хостер с IP из белого списка мобильных операторов. |
| **TCP timeout** | Клиент вообще не доходит до сервера. IP заблокирован или проблема с маршрутизацией. | Запустить `tspu_server_check.sh` — возможно порт 443 заблокирован для этого диапазона IP. |
| **Garbage data (scanner / bot)** | Кто-то подключился и отправил мусорные данные. telemt отклонил. | Нормальный фоновый шум. TCP Splice отдаёт сканеру реальный сертификат `www.bing.com` — MTProxy не детектируется. |
| **Invalid secret** | Клиент подключился с неверным секретом. | Проверить что у пользователя правильная ссылка `tg://` из API. |
| **Connection reset by client** | Клиент сам разорвал соединение до завершения хэндшейка. | Обычно Telegram переподключается после выхода из сна. Норма. |
| **TLS error** | Проблема TLS на стороне клиента. | Редкость — возможна неправильная настройка клиента или MITM-атака. |
| **Idle timeout** | Соединение простаивало 6 минут — telemt закрыл его. Telegram переподключается сразу. | Не ошибка. Отображается в секции «Background noise». |

## Что означают данные в практике

### Много ошибок `Mobile operator blocking MTProto`

```
Mobile operator blocking MTProto  (total: 104)
  128.71.112.169    70x   Russia / PJSC Vimpelcom
```

Ожидаемое поведение в мобильных сетях с жёстким DPI — Билайн, часть регионов МТС. Клиент успешно устанавливает TCP-соединение с прокси, но мобильный оператор перехватывает и обрывает MTProto-хэндшейк до завершения.

**Это не проблема сервера.** Тот же пользователь по WiFi подключается без ошибок.

### Сканеры и боты

```
Garbage data (scanner / bot)  (total: 2)
  92.118.234.182    2x   Netherlands / DataCenter
```

Порт 443 привлекает автоматические сканеры. telemt обрабатывает это корректно — сканер получает реальный TLS-ответ от `www.bing.com` и не обнаруживает MTProxy. 1–10 таких событий в час — нормальный фоновый шум.

### Idle timeout

```
Background noise: 18 events
  Idle timeout (normal — Telegram reconnects): 18
```

Telegram поддерживает постоянные соединения с прокси. Когда экран телефона выключается, соединение простаивает. Через 6 минут telemt закрывает его — Telegram переподключается при включении экрана. Это генерирует события idle timeout постоянно и является нормой.

## Установка

```bash
curl -fsSL https://raw.githubusercontent.com/it265ru/MTProxy-Auto-Installer/main/proxy-stats \
    -o /usr/local/bin/proxy-stats
chmod +x /usr/local/bin/proxy-stats
```

Или скопировать вручную с Windows:
```powershell
scp -P 8443 proxy-stats root@IP_СЕРВЕРА:/usr/local/bin/proxy-stats
```

Требования: **Python 3.6+**, `journalctl`, telemt API на `127.0.0.1:9091`

## Раздельное отслеживание пользователей

Каждый пользователь в `telemt.toml` получает свой секрет и отдельную строку в статистике.  
Чтобы давать разным людям разные ссылки и видеть каждого отдельно:

```bash
# 1. Сгенерировать новый секрет
openssl rand -hex 16

# 2. Добавить в /etc/telemt/telemt.toml (перезапуск не нужен)
[access.users]
user1 = "существующий_секрет"
alice = "новый_секрет_для_alice"
bob   = "новый_секрет_для_bob"

# 3. Получить ссылку для конкретного пользователя
curl -s http://127.0.0.1:9091/v1/users | jq -r '.data[] | select(.username=="alice") | .links.tls[0]'
```

После этого `proxy-stats` показывает каждого пользователя отдельно с его трафиком и подключениями.

## Геолокация

`proxy-stats` запрашивает `ip-api.com` чтобы показать страну и провайдера для каждого IP.  
Результаты кешируются в памяти на время сессии `watch`.

Если сервер не имеет доступа к `ip-api.com`, гео показывается как `?` — остальной вывод не затрагивается.
